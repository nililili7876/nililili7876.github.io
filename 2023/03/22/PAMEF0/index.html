进程

关于Process的理解

  1. A program in execution: 一个正在执行的程序
  2. An instance of a program running on a computer: 一个正在计算机上执行的程序的实例[这个过程就是Process]
  3. The entity that can be assigned to and executed on a processor: 能分配给处理器并有处理器执行的实体

PCB

  • Process Control Block: 进程控制块
  • 组成元素/用于描述进程: PID, state, Priority, Memory pointers

进程状态

New, Ready, Running, Blocked, Suspend, Exit

并发: 互斥和同步

0.概要

介绍实现互斥的三种方法: 信号量、管程、消息传递

两个经典的并发问题：生产者/消费者，读/写

与并发相关的关键术语

  • 原子操作: 不会被中途打断的操作，原子性可以保证并发状态的进行
    • 当多个线程并行地执行时，如果某个操作不具有原子性，那么在执行这个操作时，其他线程可能会干扰，从而导致操作结果不确定
  • 临界区：一段代码，在这段代码中进程将访问共享资源，当另外一个进程已在这段代码中运行时，这个进程就不能在这段代码中执行[一段代码，这段代码必须在任何时候只能由一个线程执行]
  • 死锁：两个或多个任务因相互等待对方释放资源而导致无限期等待的状态。在这种情况下，任务不能继续进行，因为它们都需要某种资源，但另一个任务拥有该资源并不释放它
  • 饥饿：一个任务因资源不足而不能执行的情况。这可能是由于该任务始终无法获得所需的资源，或者该任务被其他任务长期占用资源而导致资源不足。饥饿可以导致系统效率降低，并导致任务执行时间过长
  • 互斥：当一个进程在临界区访问共享资源时，其他进程不能进入该临界区访问任何共享资源的情形

1.进程间的交互

根据进程相互之间是否知道对方的存在程度，我们可以对进程间的交互方式进行分类

  1. 进程间不知道对方的存在 ==>【竞争】

    • 一个进程的结果与另一进程的活动无关

    • 竞争进程间没有任何信息交换，但一个进程的执行可能会影响到竞争进程的行为[比如说：当两个进程都期望访问同一个资源时，如果操作系统把这个资源分配给一个进程，那么另一个进程就必须等待]

    • 进程的执行时间可能会受到影响

    • 互斥[mutual exclusion]

      • 临界区：假设两个或更多的进程需要访问一个不可共享的资源，如打印机。在执行过程中，每个进程都给该 IO 设备发命令，接收状态信息，发送数据和接收数据。我们把这类资源称为临界资源 (oritical resource)，使用临界资源的那部分程序称为程序的临界区 (oritical section)。一次只允许有一个程序在临界区中。

    • 可能会出现死锁和饥饿

  2. 进程间间接知道对方的存在 ==> 【共享】

    • 一个进程的结果可能取决于另一进程的获取的信息

    • 进程的执行时间可能会受到影响

1.2 互斥的要求

  1. 必须强制实施互斥：在与相同资源或共享对象的临界区有关的所有进程中，一次只允许一个进程进入临界区。
  2. 一个在非临界区停止的进程不能干涉其他进程。
  3. 绝不允许出现需要访问临界区的进程被无限延迟的情况，即不会死锁或饥饿。
  4. 没有进程在临界区中时，任何需要进入临界区的进程必须能够立即进入。
  5. 对相关进程的执行速度和处理器的数量没有任何要求和限制。
  6. 一个进程驻留在临界区中的时间必须是有限的。

1.3 互斥的硬件支持

中断禁用

在单处理器中，并发进程不能重叠只能交替。因此为保证互斥，只需要保证一个进程不被中断即可，这可以通过系统内核的启用和禁用中断的原语

这在多处理器中没办法保证互斥

2.信号量

常用的并发机制：信号量，二元信号量，互斥量，条件变量，管程，事件标志，消息/信箱，自旋锁

定义：用于进程间传递信号的一个整数值，在信号量上只可以进行三种操作：初始化，递增和递减，这3个操作都是原子操作。二元信号量就是只取0/1的信号量。

基本原理：两个或多个进程可以通过简单信号进行合作，可以强迫一个进程在某个位置停止，直到它接收到一个特定的信号。

  • 发送信号, semSignal(s), 信号量+1，若值小于等于0，则被semWait阻塞的进程解除block, V
  • 接收信号, semWait(s)，信号量-1，若值变为负数，则block执行semwait的进程, P

为到达控制进程合作的目的对信号量进行的三个操作

  1. 一个信号量可以初始化成非负数
  2. semWait操作使信号量-1，若值变为负数，则阻塞执行semWait的进程，否则进程继续执行
  3. semSignal操作使信号量+1，若值<=0,则被semWait操作阻塞的进程解除阻塞

==>对二元信号量来说

  1. 二元信号量初始化为0/1
  2. semWaitB操作检查信号的值，若值为0，则进程执行semWaitB就会受阻。若值为1，则将值改为0，并继续执行该进程
  3. semSignal操作检查是否有任何进程在该信号上受阻。若有进程受阻，则通过semWaitB操作，受阻的进程会被唤醒；若没有进程受阻，则值设置为1。

2.2 生产者/消费者问题

问题描述：

  • 生产者：有一个或多个生产者生产某种类型的数据并放在缓冲区

  • 消费者：有一个消费者从缓冲区取数据，每次只取一项

  • 系统需要保证在任何时候只有一个主体(P或者C)可以访问缓冲区

    • 当缓存满的时候，生产者不会向其中添加数据
    • 当缓存为空时，消费者不会从中移走数据【消费者在开始之前保证生产者已经生产】
