1. Vérification de propriétés des systèmes informatiques (répartis)

sûreté et vivacité

  • sûreté: quelque chose de mauvais ne doit jamais arriver: 坏事一定不会发生
  • vivacité: quelque chose de bon doit fatalement arriver: 好的事情一定会发生
  • l’exclusion mutuelle 是 propriété sûreté
  • l’absence de famine 是 propriété vivacité

在分布式系统中，测试通常是不够的：验证往往变得无限大（由于行动的执行顺序不可预测）==> 所以需要一些方法来做这个事情

一种建模的方法/une solution: le produit synchronisé, modélisation de P(0) et F(i)

Contraintes de synchronisation(dans l’ordre P0,P1,F0,F1) représentent les actions peuvent se produire simultanément dans chaque composant

同步约束（按P0,P1,F0,F1的顺序）代表每个组件中可以同时发生的动作==>建模出来是这样的

Composition: le produit synchronise des systèmes de transitions est obtenu en itérant, à partir de l’état initial, le calcul

  • l’ensemble des transitions
  • des états alors accessibles

Système de transitions étiqueté: S = <Q,I,A,T>

  • Q: un ensemble fini d’états
  • I: sous-ensemble de initiale états
  • A: un ensemble fini d’actions ou alphabet (les étiquettes
  • T: Q X A X Q est un ensemble de transitions

复杂系统建模的方法 :

  • 分解成简单的实体。
  • 用一个过渡系统对每个实体进行建模，（有限状态机）
  • 在可接受的过渡的约束下通过乘积进行组合
  • 获得可访问状态的模型，其形式是一个过渡系统（复合）
  • 过渡系统的形式获得可访问状态的模型（复合）。

1.2 Modélisation les propriétés à vérifier

1.2.1 Une première vérification importante : l’accessibilité

Parcours exhaustif et explicite

  • propriétés de sécurité ==> propriété P = ensemble d’états

  • analyse en avant: marquer les états atteignables[1] [2] [3]…层

    当aucun mauvais état atteint 说明 P varie

  • analyse en arrière: marquer les états Faux et marquer les prédécesseurs

    当l’état initial n’est pas marque 说明 P varie

1.2.2 其他

  • interblocage : il existe un état accessible à partir duquel il n’y a plus de transitions

    有一个可访问的状态，从这个状态中没有更多的transitions

  • famine : après franchissement d’une transition (e.g. requête d’un écrivain), certaines exécutions peuvent faire qu’on ne passe plus pas un certain état (modélisant le temps d’écriture)

    在越过一个transitions（例如写作者的请求）后，某些执行可能导致错过某个状态（模拟写的时间）

  • précédence : après une transition donnée, il est certain qu’on atteindra un état donné avant un autre,

    在一个给定的transitions之后，肯定会在另一个transitions之前达到某个特定的状态。

Exemple: the train-gate

题目描述：控制进入一次只允许一列火车通过的桥梁的铁路系统（例如脆弱的桥梁）。过路控制器[ contrôleur de passage]确保在接近时只给一列火车提供通道。为了遵循这种火车的UPPAAL模型。当一列火车接近时（Appr状态），可以在10个时间单位内停止；否则就太晚了，火车必须过桥（Cross）。当它因为轨道被占用而停止（Stop）时，必须重新启动（Start）才能过桥

L’importance de cet exemple :

  1. modéliser des processus aux comportements différents

    对具有不同行为的过程进行建模

  2. synchroniser des processus sur événements

    在事件中同步进程

  3. programmer des fonctions exécutées sur les transitions (ici pour la gestion d’une file d’attente).

    在转换过程中执行的程序功能（这里用于管理一个队列）

UPPAL建模

1.TRAIN

  • train先发出Appr信号
  • 然后要么在10个时间单位停止(Stop)，要么传过去(Cross)
  • 如果停下需要重新启动(Strat)
  • 需要7到15个时间单位来重新启动

2.GATE

  • 对于第一个到达的TRAIN（空队列，在appr[]信号上通信后，它被排队并可以立即穿越（状态为TRAIN的Cross和GATE的Occ）。接下来的列车被排队并停止。
  • 当TRAIN离开SC时，它从队列中移除，闸门回到Free状态。
  • 从GATE的Free状态开始，如果队列不是空的，队列中的第一列火车可以重新开始穿越。
