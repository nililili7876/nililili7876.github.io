1. Mémoire partagée

Pour apprendre à utiliser efficacement les types de mémoire CUDA dans un programme parallèle, il faut[为了学习如何在并行程序中有效地使用CUDA内存类型，有必要] :

  • comprendre l’importance de l’efficacité de l’accès à la mémoire

  • connaître la distinction[区别] entre registres, mémoire partagée et mémoire globale

    chatGPT说：

    • 寄存器是 CPU 的一部分，它用来保存临时数据和指令。它的容量比较小，速度也比较快，但是数量有限。
    • 共享内存是 GPU 和 CPU 共同拥有的一块内存，两者都可以访问。因为它在 GPU 和 CPU 之间共享，所以速度比较快，但是容量也比较小。
    • 全局内存是 GPU 拥有的内存，但是 CPU 无法访问。它的容量相对较大，但是速度相对较慢。

    通常情况下，我们会尽量使用寄存器和共享内存来保存数据，因为它们的速度比较快。如果数据量太大，就会使用全局内存。

  • comprendre la portée[范围] et la durée de vie des variables

矩阵乘法1.0

GPU的性能分析：

所有线程都为其输入数组的元素访问全局内存，即为浮点加法进行两次内存访问（2*4 octets[字节,8bits]）

对于Jetson Nano:

  1. 4GB 64位LPDDR4内存，带宽为25.6GB/s
  2. 472 GFLOPs（每秒浮点运算）

因此，需要8*472=3776 GFLOPs来达到FLOPS的峰值。在带宽为25.6GB/s的情况下，我们达到了3.2FLOPs的极限，也就是设备最大浮点执行率的0.6%! 要想接近472 GFLOPS，必须大大减少内存访问量[accès à la mémoire]

3. 考虑线程数(M/N的大小)

  • 16*16
    • dans chaque phase，每个bloc执行2 * 256 = 512 浮点加载，全局内存 256 × (2 × 16) = 8192 次乘和加
    • 每个bloc用2 * 256 * 4o=2Ko的共享内存
    • 对于64 kb的共享内存，可能有多达32个线程块执行操作
    • 这允许多达32*512 = 16,384等待加载(每个线程2个，每个块256个线程)
  • 32*32
    • dans chaque phase，每个bloc执行2 * 1024 = 2048 浮点加载，全局内存 1024 × (2 × 32) = 65536 次乘和加
    • 每个bloc用2 * 32 * 32 * 4o=8Ko的共享内存
    • 对于64 kb的共享内存，可能有8个线程块执行操作
    • 这允许多达 32*2048 = 65 536等待加载(每个线程2个，每个块1024个线程)

4. tuiles的步骤总结

  1. Identifier une tuile de la mémoire globale qui est utilisée par de multiples threads: 识别一个被多个线程使用的全局内存tuile
  2. Charger la tuile de la mémoire globale dans la mémoire sur le SM: 将全局内存加载到共享内存中
  3. Utiliser la synchronisation des barrières pour s’assurer que tous les threads ont finis le chargement mémoire：使用synchronisation确保完成了内存加载
  4. Avoir plusieurs threds pour accéder à leurs données à partir de la mémoire partagée：让多个线程从共享内存中访问其数据
  5. Utiliser la synchronisation des barrières pour s’assurer que tous les threads ont terminé leur calcul: 使用synchronisation确保完成了计算
  6. Passer à la tuile suivante：移动到下一块tuile
